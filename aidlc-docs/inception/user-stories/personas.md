# 사용자 페르소나 (Personas)

테이블오더 서비스의 핵심 사용자 페르소나 2개를 정의합니다. 페르소나는 `requirements.md` 의 Stakeholders 와 일치하며, 모든 user story 는 두 페르소나 중 하나에 매핑됩니다.

---

## P1: Customer (고객) — "민준"

| 항목 | 내용 |
|---|---|
| **이름/맥락** | 민준, 32세, 직장인 |
| **상황** | 점심·저녁 시간에 일행 2~4명과 식당 방문, 테이블에 비치된 태블릿으로 주문 |
| **기술 친숙도** | 스마트폰·태블릿 일상 사용, 메뉴 선택과 결제는 익숙. 복잡한 인터랙션은 거부감 |
| **주요 목표(Goals)** | • 메뉴 빠르게 훑어보고 선택 후 주문 완료<br>• 일행과 합쳐 주문, 한 번에 끝내기<br>• 주문 후 상태(준비중/완료) 확인 |
| **좌절 포인트(Pain Points)** | • 로그인 절차, 회원가입<br>• 느리거나 끊기는 화면<br>• 주문이 잘못 들어갔는지 모르는 상태<br>• 모바일·태블릿에 너무 작은 버튼 |
| **사용 디바이스** | 매장 비치 태블릿 (가로/세로 양쪽 가능, 1080×1920 또는 1920×1080) |
| **사용 환경** | 매장 내 Wi-Fi, 짧은 체류 시간 (10~60분) |
| **세션 지속** | 한 끼 식사 단위(테이블 세션) — 첫 주문~이용 완료까지 |
| **성공 정의** | "메뉴 보고 → 담고 → 주문 확정 → 완료" 5분 이내 끝나면 성공 |

### 핵심 시나리오 흐름
1. 태블릿 자동 로그인된 상태로 매장 메뉴 화면 진입
2. 카테고리 탭으로 메뉴 탐색, 카드 터치로 상세 확인
3. 메뉴를 장바구니에 담고 수량 조절
4. 주문 확정 → 주문번호 확인 → 5초 후 메뉴 화면 복귀
5. (필요 시) "주문 내역" 진입하여 현재 세션 주문 확인

## P2: Admin (매장 운영자) — "지우"

| 항목 | 내용 |
|---|---|
| **이름/맥락** | 지우, 50대, 매장 운영자(점주 또는 매장 매니저) |
| **상황** | 매장 카운터/주방 근처에서 PC 또는 태블릿으로 주문 모니터링·테이블 관리. 점심·저녁 피크에 동시 다발 주문 처리 |
| **기술 친숙도** | POS·관리자 도구 사용 경험, 새로운 시스템 학습 의지 있음. 복잡한 설정은 부담 |
| **주요 목표(Goals)** | • 들어오는 주문을 한 화면에서 실시간으로 파악<br>• 테이블별 주문·총액·상태를 빠르게 확인<br>• 잘못 들어온 주문 즉시 삭제<br>• 손님 이용 완료 시 깔끔하게 세션 정리<br>• 메뉴 가격·구성 자유롭게 수정 |
| **좌절 포인트(Pain Points)** | • 주문이 늦게 보이는 것<br>• 어느 테이블의 주문인지 헷갈리는 것<br>• 메뉴 수정 후 고객 화면 반영 지연<br>• 복잡한 권한·계정 관리 |
| **사용 디바이스** | 매장 카운터 PC (≥ 1366×768) 또는 태블릿 |
| **사용 환경** | 매장 내 안정적 네트워크, 16시간 연속 세션 |
| **세션 지속** | 영업일 1회 로그인 → 16시간 세션 |
| **성공 정의** | "주문 들어옴 → 2초 이내 화면에 표시 → 상태 변경 → 완료 처리" 흐름이 끊김 없이 진행 |

### 핵심 시나리오 흐름
1. 영업 시작 시 매장 ID + 계정으로 로그인
2. 그리드 대시보드에서 테이블별 카드로 실시간 주문 확인
3. 신규 주문 들어오면 시각적 강조, 카드 클릭으로 상세 확인
4. 주문 상태 변경(대기중→준비중→완료)
5. 잘못된 주문은 삭제(soft-delete) 처리
6. 손님 식사 종료 시 "이용 완료" 처리 → 다음 손님 받을 준비
7. 메뉴/카테고리/이미지는 별도 화면에서 CRUD

## 페르소나 사용 가이드

| 의사결정 영역 | 우선순위 기준 |
|---|---|
| UI 응답 속도 vs 기능 풍부함 | **응답 속도 우선** (Customer/Admin 둘 다 시간 민감) |
| 학습 곡선 | **Customer는 zero-learning, Admin은 30분 학습 허용** |
| 에러 발생 시 | **Customer는 단순 메시지 + 재시도, Admin은 상세 + 로그** |
| 단축키·고급 기능 | **Admin에만** 적용 검토 (Customer는 단순 유지) |

## System / Backend (페르소나 없음 — 시스템 행위)

`requirements.md` 의 FR-SYS-* 항목은 직접 사용자 인터랙션이 아닌 시스템 자동 행위로, `stories.md` 의 **부록 섹션 "System Behaviors"** 에서 별도 트래킹합니다. 이 항목은 페르소나 매핑 없이 시스템 책임으로 기록됩니다.
